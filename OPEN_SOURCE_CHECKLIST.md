# Open Source Checklist

Before publishing:

- [x] Choose a license.
- [x] Add `LICENSE`.
- [x] Decide the GitHub owner and repository name.
- [ ] Initialize this folder as its own git repository, or move it out of the parent repo.
- [ ] Confirm `notes/`, `dist/`, `mockups/`, and local state are not tracked.
- [ ] Run `npm run doctor`.
- [ ] Create the GitHub repository.
- [ ] Push the initial commit.
- [ ] Add screenshots or a short demo GIF after private content is removed.

Recommended repository description:

```text
Always-on-top desktop learning companion for Codex issue-learning sessions.
```
