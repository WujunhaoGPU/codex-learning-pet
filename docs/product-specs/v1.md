# v1 Product Spec

## Goal

Keep one current Codex learning step visible on the desktop while the user works
through an issue.

## User

A Codex user learning from issues, bugs, or repo debugging sessions.

## Core Path

1. User starts the pet app.
2. App connects to `codex app-server`.
3. App resumes or watches the latest thread in this project.
4. Codex events update local learning state.
5. The always-on-top pet shows one current action.
6. The panel shows route progress, status, current goal, current step, next step, and latest knowledge.
7. The user can open or save knowledge notes for the current thread.

## Acceptance Criteria

- Pet window is always on top.
- App connects to app-server and receives thread / turn / item events.
- App persists state under macOS Application Support.
- App restores state after restart.
- UI shows route and status views.
- Route progress does not jump across many steps from incidental text matches.
- Knowledge notes are grouped by date and thread.
- A three-turn issue-learning flow leaves enough state to continue without reading chat history.
- A real run can be checked with `npm run doctor -- --current-state`.

## Non-Goals

- Multi-user sync.
- Plugin marketplace.
- Complex pet animation.
- Replacing the Codex desktop app.
